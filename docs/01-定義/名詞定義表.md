## 一、憲法級名詞（具憲法效力，不得私自重定義）

| 名詞 | 一句話定義 | 出處 |
|---|---|---|
| Claim（斷言） | Provider 對輸入回報的結構化陳述；**有出處的意見，不是事實** | 憲法 §3 |
| Label（標註） | 經 Human Review 批准的 Claim；唯一能進入 Dataset 的東西 | 憲法 §3、D7 |
| Vision Provider | 任何「輸入影像→回傳 Claim」的元件，本地雲端、可否訓練一視同仁 | 憲法 §3 |
| Provider Capability | Provider 機器可讀的能力自我聲明；聲明是履歷，試鏡才是面試 | 憲法 §3、PR5 |
| Teacher（老師） | **角色非身分**：該次調用中產生 Claim／診斷的 Provider | 憲法 §3 |
| Student（學生） | **角色非身分**：被訓練以供部署的 Provider（如 YOLO） | 憲法 §3 |
| Vision Orchestrator | 唯一有權決定「哪個 Provider 何時處理什麼」的核心元件；永不解讀像素 | 憲法 §3、§4 |
| Confidence（信心值） | Provider 自報分數；跨 Provider 比較原始值＝違憲；未校準一律人審 | 憲法 §3 |
| Calibration（校準） | 以 Golden Dataset 將信心值映射至實測精確率的程序 | 憲法 §3 |
| Decision（決策） | Orchestrator 的不可變紀錄（政策、選擇、成本、結果），必可重放 | 憲法 §3、D3 |
| Evidence（證據） | 使 Decision／Claim 可被人複核的最小材料集 | 憲法 §3、A6 |
| Dataset（資料集） | 版本化資產：雜湊定址媒體＋Label＋切分＋Taxonomy＋manifest | 憲法 §3 |
| Golden Dataset（黃金集） | 純人工基準；三鐵律（永不訓練、永不被機器覆寫、最終裁決）；四職能（仲裁／校準／試鏡／漂移基準） | 憲法 §3、D8 |
| Human Review（人工審核） | Claim → Label 的唯一合法程序；審核行為本身沉澱為記憶 | 憲法 §3、D7 |
| Project Memory（專案記憶） | 專案內自動累積的結構化事實；屬於專案、存於專案 | 憲法 §3、§7 |
| Knowledge Base（知識庫) | 產品層通用知識；「世界的常識」vs 記憶的「你專案的事實」 | 憲法 §3 |
| Application Pipeline | 消費 Label／Claim 產生業務產物的下游流程；userland | 憲法 §3、F8 |
| Cost Engine | 計量、預測、預算執行的確定性子系統；量與擋，不決策 | 憲法 §3、§6 |
| Budget（預算） | 硬約束的資源上限；超支即停＋上報 | 憲法 §3、C2 |
| Escalation（升級） | 移交更強 Provider 或人工；人工是最貴的 Provider | 憲法 §3、C4/C5 |
| 試鏡（Teacher Audition） | Provider 以本專案 Golden Dataset 實測取得路由資格的程序 | PR5 |
| 確定性保留區 | 帳、版本、裁決路徑；LLM 與機率性元件不得進入 | A5、D11 |

## 二、工作用語（協作慣用語，效力=其出處文件）

| 名詞 | 一句話定義 | 出處 |
|---|---|---|
| 血統（Provenance） | 每筆標註／Claim 的完整出處鏈（誰的哪一版＋提示＋是否人改） | R1 §9、憲法 PR6 |
| 五道防線 | Auto Label 治理：信心分流／血統／黃金集／入庫閘門／盲審稽核 | R1 §9（R2 確認保留） |
| 入庫閘門（Approval Gate） | 資料合併為新版本前的檢查關卡（統計閘門先行） | R1 §9.3 |
| 盲審抽樣／蜜罐 | 抽驗已通過標註（不標示來源）／混入已知答案測審核者狀態 | R1 §9.3、R2 9.3（含衰減） |
| 聚類網格審核 | 按類別＋視覺相似聚類的批次審核 UX；未來所有標註都是審核 | R1 §9.4 |
| 主動學習 | 優先標模型最不確定／最沒看過的資料；反向建議「這些不用標」 | R1 §9.5 |
| 開放詞彙 Taxonomy | 類別＝文字描述＋正反例＋提示詞的概念卡，非封閉整數 ID | R2 問題二、憲法附錄 #3 |
| Staging（隔離區） | 審核完成、尚未通過閘門的資料暫存區 | R1 §9.3 |
| 內容雜湊定址 | 媒體以 SHA-256 為身分儲存，天然去重 | R1 §5.4 |
| Embedding 基礎設施 | 匯入即向量化；支撐去重／搜尋／錯標偵測／主動學習 | R2 問題二翻轉三 |
| TTFV | Time To First Value：安裝→親眼看到自己的圖被看懂；目標 10 分鐘 | R2 9.4 |
| 下一步引擎 | 由專案狀態推導唯一下一步的確定性導引 | R1 §2.4／§6.2 |
| 白話翻譯層 | 結論先行、證據隨後、術語墊底的全產品呈現紀律 | R1 §11 |
| 部署包／一致性驗證 | 模型＋前後處理參數＋範例；匯出後與原模型比對輸出 | R1 §13.4 |
| 看懂→整理→鑄造→應用 | R3 基準的四迴圈骨架（Understand／Curate／Distill／Apply） | R2 10.6 |
| M0／M1／M2／M3 | 分期：資料工房／鑄造廠／品質引擎／平台 | R2 10.6、憲法 §9 |
| kernel／userland | OS 隱喻：核心（調度治理資產）／外圍（GUI、管線、Copilot） | 憲法前言 |
| Rule of Two | 抽象須有兩個真實實作方可固化 | A10、D12 |
| 規格票 | 發包給 Codex 的任務單：目標／範圍／介面／驗收／審查層級／憲法／禁區，缺欄退票 | 分工協議 §3 |
| 審查分級（L0/L1/L2） | L0 免審（CI 綠即併）／L1 事後抽審（信任衰減）／L2 必審（守門路徑）；新承包者首票全審 | 分工協議 §3-2 |
| 帳本（Ledgers） | DecisionRecord/DecisionOutcome、CostEntry（estimate/actual 配對）、ReviewEvent（含盲審/蜜罐 context）、DatasetVersionManifest、GoldenSetEntry 五組 append-only 記錄 | ADR-0004、`contracts/ledgers.py` |
| 紅線 | Codex 不得觸碰的區域與行為，CI 強制 | 分工協議 §4 |
