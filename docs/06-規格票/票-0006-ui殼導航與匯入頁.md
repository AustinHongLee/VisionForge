# 票-0006：ui 殼導航（核心迴圈）＋匯入頁骨架

| 項目 | 內容 |
|---|---|
| 狀態 | 🟨 已發出（2026-07-08，批次雙軌） |
| 承包 | Codex（Builder） |
| 審查層級 | **L0 免審**（ui/ 白名單；CI 全綠＋D20 聲明即由決策者直接合併，協議 §3-2） |
| 政策依據 | R2 §10.6（🟩 定案）新核心迴圈＝開發基準；型別消費票-0004（contracts.generated） |

## 【目標】

把 Renderer 的 placeholder 換成反映**新核心迴圈**的殼導航——「看懂 Understand → 整理 Curate → 鑄造 Distill → 應用 Apply」（R2 10.6），並在「看懂」站落一個**匯入頁骨架**：drag-drop 區＋縮圖網格。網格資料**先用 stub**（型別取自 `contracts.generated`），真正接後端另票。**純前端、純展示，不接 Python、不動安全設定。**

## 【範圍】（白名單）

`ui/src/renderer/**` 為主：可新增/改 `App.tsx`、新增元件於 `ui/src/renderer/src/components/`、stub 資料於 `ui/src/renderer/src/data/`、樣式 `styles.css`、對應測試（vitest）。可微調 `ui/src/shared/contracts.ts` barrel 以**再匯出**所需型別（僅 re-export，不得新增契約定義）。

**不得動**：`ui/src/main/**`、`ui/src/preload/**`（票-0002 安全殼）、`index.html` 的 CSP、`.dependency-cruiser.cjs` 規則（除非新增而非放寬）、`electron-builder.yml`、`contracts.generated.ts`（生成物）、`core/**`、`app/**`。

## 【介面】（骨架規格——結構凍結，樣式與細節自由）

1. **殼導航**：四站 `看懂 / 整理 / 鑄造 / 應用`（英標 Understand/Curate/Distill/Apply），以 React state 切換 active 站。「鑄造」「應用」本票只放「施工中」佔位面板。
2. **匯入頁**（置於「看懂」站）：
   - `DropZone`：drag-over 視覺回饋；`onDrop` 把拖入檔名列成「待匯入（尚未接後端）」清單（**純前端 state，不讀檔內容、不呼叫 fs／electron**）。
   - `ThumbnailGrid`：渲染一組 `MediaRecord` 型別的資料為卡片，每卡顯示 `format`、`{width_px}×{height_px}`、`media_hash` 前 8 碼；縮圖以 CSS 佔位色塊（**不得引用遠端圖片**，維持 CSP）。
3. **stub 資料**：`ui/src/renderer/src/data/mediaStub.ts` 匯出 `SAMPLE_MEDIA`，型別**必須**來自生成物：
   ```ts
   import type { MediaRecord } from "../../../shared/contracts.generated";
   export const SAMPLE_MEDIA: MediaRecord[] = [ /* 3–6 筆合理假資料 */ ];
   ```
   （藉此 dogfood 票-0004 的型別；欄位須符合 `MediaRecord` 形狀：`schema_version:"1.0"`、`media_hash`、`width_px`、`height_px`、`format`、`byte_size`、`imported_at`(ISO 字串)、`source:{kind,detail}`、`exif_normalized`。）
4. **桌面能力一律經 Bridge**：如需版本／通知走既有 `window`-bridge，**禁止直接 `import electron`**（A3，depcruise 強制）。

## 【工作項】

1. 重構 `App.tsx` 為殼＋四站導航；保留既有版本顯示（`getAppVersion`）於殼頁首或「看懂」站。
2. 新增 `DropZone`、`ThumbnailGrid` 元件與 `mediaStub.ts`。
3. 「看懂」站組合 DropZone＋ThumbnailGrid（後者吃 `SAMPLE_MEDIA`）。
4. 樣式：沿用既有 `styles.css` 風格，網格 responsive；不引第三方 UI 庫（除非已在 package.json）。
5. 測試（vitest＋RTL）：見驗收。

## 【驗收】（測試清單＝完成的唯一定義）

1. 殼渲染四站標籤；點擊切換 active 站，「鑄造/應用」顯示「施工中」佔位。
2. 「看懂」站渲染 DropZone 與 ThumbnailGrid；Grid 卡片數＝`SAMPLE_MEDIA.length`，且每卡顯示 format 與 `width×height`。
3. DropZone 模擬 drop 檔案 → 檔名出現在「待匯入」清單（純前端，不觸發任何 fs／IPC）。
4. `SAMPLE_MEDIA` 以 `MediaRecord`（生成型別）標註，`tsc --noEmit` 綠（證明型別相容＝票-0004 生效）。
5. `pnpm typecheck`、`pnpm lint`、`pnpm test`、`pnpm depcruise`、`pnpm build` 全綠。
6. 無 `import electron`（renderer）；CSP 未放寬；`ui/src/main`、`ui/src/preload` 零改動。

## 【憲法】

R2 §10.6（核心迴圈＝資訊架構基準）、A3／D13（Renderer 經 Bridge，禁直連 electron；依賴方向不反向）、票-0002 安全殼不得回退（CSP／導航鎖／urlPolicy 保持）、A10（不預建「整理/鑄造/應用」的實體功能，本票只立骨架與匯入頁）、唯一事實來源（型別取自 contracts.generated，不手寫平行定義）。

## 【禁區】

不得直接 `import electron` 於 renderer；不得放寬 CSP／導航白名單／depcruise 規則；不得動 `ui/src/main`、`ui/src/preload`、`electron-builder.yml`；不得手寫契約型別（一律用生成物）；不得引遠端圖片資源；不得接 Python／fs／真實匯入（那是後端接線票，另發）；規格衝突或缺件 → 停手回報，不腦補。
