## 【目標】

把已審 Label 匯出成 **YOLO** 或 **COCO** 資料集檔。流程：`build_version`（core，凍成版本快照）→ 依 manifest 把影像（blob）與標註寫到 `project.root/exports/{version_id}/`。class 索引由專案 Taxonomy 決定。**版本化/內容定址由 core 保證，app 只做格式轉換與檔案 IO。**

## 【範圍】（白名單）

`app/**`。新增 `app/src/visionforge_app/export/`（YOLO/COCO writer）、於 `app/src/visionforge_app/api/app.py` 加 `POST /export`；測試於 `app/tests/`。

**不得動**：`core/**`、`ui/**`、`.github/**`、`scripts/**`、根/providers `pyproject.toml`。

## 【介面】（已凍結——只實作本體）

```python
# app/src/visionforge_app/export/dataset.py
from dataclasses import dataclass
from pathlib import Path
from typing import Literal
from visionforge_core.storage import Project

@dataclass(frozen=True)
class ExportOutcome:
    version_id: str
    fmt: Literal["yolo", "coco"]
    out_dir: Path
    image_count: int
    label_count: int
    class_names: tuple[str, ...]   # index → concept.raw_text

def export_dataset(
    project: Project,
    fmt: Literal["yolo", "coco"],
    *,
    version_id: str,          # 傳給 build_version
    created_at: datetime,
    out_root: Path | None = None,   # 預設 project.root/"exports"
) -> ExportOutcome: ...
```

**消費 core（不得改）**：`dataset.build_version(project, version_id=, created_at=) -> DatasetVersionManifest`；`project.manifests.get`、`project.labels.get`、`project.taxonomy.list`、`project.blobs.find`、`project.media.get`。

## 【工作項】

1. `build_version(...)` 建版本快照（無 Label → 讓其 `ValueError` 冒出成 409/422，不自建空集）。
2. **class 映射**：`project.taxonomy.list()` 依序（created_at, node_id）→ index 0..N；`class_names[i]=node.raw_text`；`taxonomy_node_id → index` 對照。
3. **YOLO**：`out_dir/images/{split}/` 放影像（由 `blobs.find(media_hash)` 複製，副檔名依 `media.format`）；`out_dir/labels/{split}/{stem}.txt` 每行 `class_id cx cy w h`（由 BBox 正規化座標換算：cx=(x1+x2)/2、cy=(y1+y2)/2、w=x2−x1、h=y2−y1）；`out_dir/data.yaml` 列 `names`/`nc`/`train`/`val`。
4. **COCO**：`out_dir/annotations.json`（images/annotations/categories；bbox 用像素 `[x,y,w,h]`＝正規化×寬高；categories 依 class 映射）；影像同樣複製到 `out_dir/images/`。
5. **只處理 `final_geometry.type=="bbox"` 的 Label**（fixture＝bbox；polygon/mask 先略過，記入回傳計數差）。
6. `POST /export` body `{fmt}` → 產 `version_id`(ULID)＋now → `export_dataset` → 回 `ExportOutcome`（out_dir 字串化）。

## 【驗收】（測試清單＝完成的唯一定義）

1. 匯入圖→process→approve 幾筆後 `export_dataset(fmt="yolo")` → `out_dir` 存在；`data.yaml` 的 `names` ＝ class_names；每張圖對應一個 `.txt`，行數＝該圖 bbox Label 數；座標為合法 YOLO 正規化值（0–1）。
2. `fmt="coco"` → `annotations.json` 合法 JSON，`images`/`annotations`/`categories` 數量正確，bbox 為像素值。
3. class 映射確定性：同 Taxonomy → 同 index 順序（連跑兩次一致）。
4. 影像確實複製到 `images/`（檔案存在、位元組＝blob）。
5. 無已審 Label → `POST /export` 回 409/422（core `ValueError`），不建空匯出。
6. 版本快照有落庫：`project.manifests.get(version_id)` 可讀回。
7. `python scripts/check_forbidden_imports.py`、`ruff check app`、`pytest app/tests` 全綠。

## 【憲法】

R2 §9.6（M0 終點）、D5（版本＝清單、內容定址——版本由 core build_version 保證，app 不得自組 manifest）、D13（app→core）、A5（class 映射/切分確定性）、A7/D16（開放格式 YOLO/COCO）、D14（無資料結構化報錯）。

## 【禁區】

不得改 `core/**`（含不得自行組 DatasetVersionManifest——一律走 `build_version`）；不得動 `ui/**`、`.github/**`、`scripts/**`、`pyproject.toml`（根/providers）；不得引入 Pillow 以外影像相依（複製 blob 不需解碼）；規格衝突或缺件 → 停手回報，不腦補。
