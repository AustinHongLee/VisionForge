## 【工作項】（依 ADR-0006）

1. **解碼＋驗證先行**：Pillow 開圖；格式不在 `MediaFormat` 封閉集 → `UnsupportedMediaFormatError`；多幀 → `MultiFrameMediaError`；無法解碼 → `MediaDecodeError`。**驗證未過前不寫任何 blob／帳本。**
2. **EXIF 正規化（Policy B）**：Orientation 缺失或 =1 → 原始位元組原封入庫；Orientation 2..8 → `ImageOps.exif_transpose` 轉正、重編碼為同格式後入庫。兩者 `exif_normalized=True`；`width_px/height_px` 取轉正後尺寸。
3. **入庫**：`media_hash = project.blobs.put(入庫位元組, format)`。
4. **冪等**：`project.media.exists(media_hash)` 為真 → 回傳既有 `record`、`deduplicated=True`，不重複 `add`、不重生縮圖。
5. **縮圖**：由已正規化影像生成，最長邊 ≤ `thumbnail_max_edge`、保持長寬比、JPEG q80，原子寫 `project.root/"media"/"thumbs"/{media_hash}.jpg`；已存在則跳過。
6. **落戶籍**：組 `MediaRecord`（`imported_at` 用 tz-aware 現在時刻、`source` 用傳入值、`exif_normalized=True`）→ `project.media.add(record)`。回傳 `ImportOutcome(media_hash, record, deduplicated=False)`。

## 【驗收】（測試清單＝完成的唯一定義；golden-file 為主）

1. orientation=1／無 EXIF 之 JPEG：入庫位元組＝原檔、`media_hash=sha256(原檔)`、`exif_normalized=True`、尺寸正確；record 入 media 表、blob 入 `media/blobs`、縮圖入 `media/thumbs/{hash}.jpg`。
2. orientation=6 之 JPEG：入庫位元組≠原檔；重新讀回 blob 後方向已剝除、像素已轉正、尺寸為轉正後值。
3. **冪等**：同一 bytes 匯入兩次 → 第二次 `deduplicated=True`、`media.count()` 不增、不拋 `ConflictError`。
4. 各支援格式 round-trip：png／webp／bmp／tiff（orientation=1）各一，`format` 映射正確、blob 副檔名正確。
5. 不支援格式（如 gif 或非影像 bytes）→ `UnsupportedMediaFormatError`；media／blob **零寫入**。
6. 多幀輸入（多頁 tiff 或動畫）→ `MultiFrameMediaError`；零寫入。
7. 損壞位元組（截斷／亂數）→ `MediaDecodeError`；零寫入。
8. 縮圖最長邊 ≤ `thumbnail_max_edge` 且長寬比保持。
9. **golden-file 回歸**：至少兩組 `(原檔, 期望 media_hash)` 釘死於測試資料。注意：orientation=1 分支雜湊穩定可釘死；**轉正分支雜湊依 Pillow 版本，改測「轉正後解碼的像素／尺寸」而非釘死 hash**（ADR-0006 已知取捨）。
10. `test_importer_contract.py` 維持綠（凍結介面未破）。
11. `python scripts/check_forbidden_imports.py` 綠、`ruff check app` 綠、`pytest app/tests` 綠。
