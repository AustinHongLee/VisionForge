# 票-0014：app 校準接線——回填信賴度、觸發重校準

| 項目 | 內容 |
|---|---|
| 狀態 | 🟨 已發出（2026-07-08，審核飛輪 #5 的 app 側） |
| 承包 | Codex（Builder） |
| 審查層級 | **L0 免審**（app/ 白名單；校準統計已由 core 封裝；CI 全綠＋D20 即由決策者合併） |
| 政策依據 | ADR-0010 #5、ADR-0007；消費 core `calibration.apply_latest / recalibrate`（已合併） |

## 【目標】

把已閉合的校準飛輪接進 app：推論產生的 claim 套用**最新校準快照**回填 `calibrated`／`reliability`（讓「看懂」站的框從全灰變出綠/琥珀/灰）；加 `POST /recalibrate` 由已審結果重算並存快照。**統計都在 core，app 只負責在對的時機呼叫 apply_latest／recalibrate。**

## 【範圍】（白名單）

`app/**`。改 `app/src/visionforge_app/processing/run.py`（process_media 回填）、`app/src/visionforge_app/api/app.py`（/infer 回填＋新增 /recalibrate）；測試於 `app/tests/`。

**不得動**：`core/**`、`ui/**`、`.github/**`、`scripts/**`、根/providers `pyproject.toml`。

## 【介面】（消費 core，不得改 core 簽名）

```python
from visionforge_core.calibration import apply_latest, recalibrate
# apply_latest(project, confidence, concept_key) -> Confidence
# recalibrate(project, *, calibration_id, created_at, golden_manifest_ref, ...) -> CalibrationSnapshot | None
```

Claim 為 frozen——回填用 `claim.model_copy(update={"confidence": apply_latest(project, claim.confidence, claim.concept.raw_text)})`。

| 方法 | 路徑 | 請求 | 回應 |
|---|---|---|---|
| POST | `/recalibrate` | — | `CalibrationSnapshot`（有觀測）／`204`（無已審結果，不建空快照） |

## 【工作項】

1. **process_media 回填**：`provider.infer` 得 claims 後、`record_inference_run` 之前，對每個 claim 以 `apply_latest` 回填 confidence（`concept_key=claim.concept.raw_text`）→ 存入的 claim 即帶當時信賴度。
2. **/infer 回填**：無狀態預覽端點回傳前，一樣用 `apply_latest` 回填每個 claim（畫面即時反映最新校準）。
3. **POST /recalibrate**：產 `calibration_id`／`golden_manifest_ref`（ULID）與 `created_at`（now）→ 呼叫 `recalibrate(...)`；有快照回 200＋快照，`None` 回 204。
4. 不改變無快照時的行為（reliability 維持 none＝灰）。

## 【驗收】（測試清單＝完成的唯一定義；TestClient）

1. 尚無快照時 `process_media`／`/infer` 的 claim `reliability == "none"`、`calibrated is None`（現況不回退）。
2. 匯入圖→process→審核（approve 幾筆、reject 幾筆，可用低 n_min/n_high 由端點預設或測試路徑）→ `POST /recalibrate` → 200 回快照。
3. 重校準後再 `/infer` 同概念 → claim `reliability` 依快照變（該概念樣本足即 `high`/`low`），`calibration_ref` 指向快照。
4. 無任何已審結果時 `POST /recalibrate` → 204（不建空快照）。
5. `process_media` 回填後存入的 claim 重讀回帶正確 confidence（determinism 注入 now/id 不變）。
6. `python scripts/check_forbidden_imports.py`、`ruff check app`、`pytest app/tests` 全綠。

## 【憲法】

ADR-0010 #5／ADR-0007（校準）、§3（分流只讀 calibrated；none/low 一律人審——本票只回填，不改分流規則）、D13（app→core）、A5（注入 now/id）、D14（/recalibrate 無資料回 204 而非假快照）。

## 【禁區】

不得改 `core/**`（含不得自行做校準統計——一律走 `apply_latest`／`recalibrate`）；不得動 `ui/**`、`.github/**`、`scripts/**`、`pyproject.toml`（根/providers）；不得在無快照時偽造 reliability；規格衝突或缺件 → 停手回報，不腦補。
