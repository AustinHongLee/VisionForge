## 【目標】

在「整理 Curate」站落審核 UX：列出待審 claim（依概念分組）、縮圖上疊出框、逐張 **批准／否決**，並一鍵 **重新校準**。這是 M0 資料飛輪的最後一塊——審完 → Label → 校準 → 「看懂」站的框開始變出綠/琥珀/灰。**純 renderer、經既有 bridge 取 API URL、不碰安全殼。**

## 【範圍】（白名單）

`ui/src/renderer/**`。擴充 `src/api/client.ts`（審核端點）、新增「整理」站元件、改導航接上（票-0006 的「整理」站目前是佔位）、對應測試。**唯讀**用 `shared/contracts.generated` 與 `window.bridge`。

**不得動**：`ui/src/main/**`、`ui/src/preload/**`、`ui/src/shared/**`、`index.html`/CSP、`app/**`、`core/**`、`providers/**`、`.github/**`、`scripts/**`。

## 【介面】（結構規格——行為照此，樣式/版面自由或依 Figma）

API client 擴充（base URL 經 `window.bridge.getApiBaseUrl()`）：

```ts
import type { Claim, Label, CalibrationSnapshot } from "../../../shared/contracts.generated";

interface PendingItem { claim: Claim; run_ref: string; media_hash: string; }

reviewPending(): Promise<PendingItem[]>;                       // GET /review/pending
approveClaim(body: { claim_id: string; run_ref: string; media_hash: string;
                     reviewer: string }): Promise<Label>;      // POST /review/approve
rejectClaim(body: { claim_id: string; run_ref: string; media_hash: string;
                    reviewer: string }): Promise<{ event_id: string; to_status: string }>; // POST /review/reject
recalibrate(): Promise<CalibrationSnapshot | null>;            // POST /recalibrate（204→null）
```

**畫面行為**：
1. 「整理」站載入 → `reviewPending()` → 依 `claim.concept.raw_text` **分組**顯示；空佇列顯示空狀態。
2. 每張待審卡：該 media 縮圖（`thumbnailUrl`）＋疊出該 claim 的 BBox（**復用票-0011 的 BoxOverlay**）＋顯示概念與 `confidence.raw`＋**批准／否決**兩鈕。
3. 批准 → `approveClaim(...)`；否決 → `rejectClaim(...)`；完成後該卡消失（重載 pending）。`reviewer` 先用固定值（如 `"local-user"`，無登入）。
4. 頂部「重新校準」鈕 → `recalibrate()` → 顯示結果（如「已用已審結果重校準」或「尚無已審結果」對應 204）。
5. 只處理 `geometry.type === "bbox"` 的 claim（fixture 只出 bbox；其他略過）。

## 【工作項】

2. 「整理」站元件：分組清單＋審核卡（縮圖＋框＋概念＋兩鈕）＋重新校準鈕。
3. 導航把「整理」站從佔位接上真內容。
4. 測試（vitest＋RTL，`window.bridge`／`fetch` 以 mock）。

## 【驗收】（測試清單＝完成的唯一定義；mock bridge/fetch）

1. `reviewPending` 回 3 筆（2 concept）→ 依概念分成 2 組、共 3 張卡。
2. 每張卡顯示概念與 raw 信心；縮圖上有框（BoxOverlay，座標比例正確）。
3. 點「批准」→ 呼叫 `approveClaim`（帶該卡 claim_id/run_ref/media_hash）→ 之後重載 pending。
4. 點「否決」→ 呼叫 `rejectClaim` → 重載 pending。
5. 點「重新校準」→ 呼叫 `recalibrate`；204→顯示「尚無已審結果」訊息、不崩。
6. API 失敗（fetch 500 / getApiBaseUrl reject）→ 錯誤狀態，不白畫面。
7. `pnpm typecheck`、`pnpm lint`、`pnpm test`、`pnpm depcruise`、`pnpm build` 全綠；renderer 無 `import electron`。
