## 背景

R2 §9.2 判為【嚴重】內傷：R1 想用「黃金集上的信心–精確率曲線逐類自動校準」門檻，但黃金集初期每類約 20 張。**20 個樣本逐類獨立擬合出的 precision-confidence 曲線是統計噪音**——校準出的門檻披著「數據驅動」外衣，卻可能比拍腦袋更危險。若不處理，分流引擎會用假門檻把低品質草稿放進「快速確認」佇列，污染資料飛輪的源頭。

契約層已預先為此挖好插槽：`Confidence` 分 `raw` 與 `calibrated`，且憲法規定「分流只准讀 `calibrated`；`reliability` 為 `none`/`low` 一律人審」（見 `contracts/claims.py`）。缺的是**產生 `calibrated`／`reliability`／`calibration_ref` 的引擎本身**，以及它所引用的校準快照契約。這是 kernel、統計不外包（分工協議 §2）。

## 決策

採用**經驗貝氏式的「全域先驗＋逐類收縮（shrinkage）」**估計每類信心門檻，並為每一類**聲明信賴度分級**，數據不足時退回保守全域門檻並強制人審。具體：

1. **門檻估計（收縮，非逐類獨立擬合）**：先由**全部**黃金集樣本擬合一條全域 precision-confidence 曲線，取達到目標精確率 `P*` 的全域門檻 `t_global`。每類門檻為
   `t_c = λ_c · t_global + (1 − λ_c) · t_c_raw`，
   其中 `t_c_raw` 是該類獨立門檻、`λ_c = N0 / (N0 + n_c)` 為收縮權重（`n_c`＝該類有效樣本、`N0`＝先驗強度常數）。**樣本越少越向全域收縮**，噪音自然被壓住。

2. **信賴度分級（none/low/high，對映 `Reliability`）**，依有效樣本量 `n_c`：
   - `n_c < N_min`（預設 30）→ **none**：不輸出逐類 `calibrated`，該類 Claim 用保守全域門檻、`reliability=none`、**強制 `queued_manual`**；快速確認佇列對該類縮到零。
   - `N_min ≤ n_c < N_high`（預設 200）→ **low**：輸出收縮 `calibrated`，但依憲法**仍一律人審**（`queued_detail`）；UI 明示「校準信賴度：低（黃金集樣本不足）」。
   - `n_c ≥ N_high` → **high**：`calibrated` 可信，`calibrated ≥ t_c` 者方可進 `queued_fast`（快速確認）。

3. **校準快照為一等契約與帳本記錄**：每次校準產出一份不可變 `CalibrationSnapshot`（含方法、全域先驗、逐類門檻/λ/n/reliability、所依據的黃金集版本 ref），以 ULID 定址，寫入 append-only 帳本。Claim 的 `confidence.calibration_ref` 指向它——**信賴度是可驗證的**（P5：不可驗證的能力視同不存在）。

4. **確定性**：給定同一黃金集版本與同一組參數（`P*`、`N0`、`N_min`、`N_high`、方法版本），輸出的快照逐位元相同（A5）。方法版本以 `method="shrinkage_eb_v1"` 釘死，改演算法＝出新版本字串。

擬新增契約（`core/contracts/`，本 ADR 採納後由 Architect 實作）：

```python
class ClassCalibration(StrictModel):
    concept_key: str            # 類身分：taxonomy_node_id 優先，否則正規化 raw_text

    shrinkage_weight: float     # λ_c ∈ [0,1]

    golden_manifest_ref: Ulid   # 血統：本次校準依據的黃金集版本

```

## 理由

- **直擊 R2 9.2 的病灶**：收縮讓「小樣本類」自動借用全域資訊而不是被自己的噪音誤導；這是處理小樣本分類問題的教科書做法（Stein 悖論／經驗貝氏），比逐類獨立擬合穩健得多。
- **與憲法既有不變量咬合**：`reliability`＝none/low 一律人審已寫死在 `Confidence`／`ReviewStatus`；本引擎只是產生這些值的來源，不需改憲法。
- **謙虛可泛化**（P5、R2 9.2 結語）：「凡自動校準都必須聲明信賴度、數據不足時退保守」被落成 none 分級 + 全域退路 + 快速佇列縮窄，而非隱性地信任壞門檻。
- **血統與可重現**：快照引用黃金集版本、方法釘版、確定性輸出——校準結果本身可被審計與回溯（P4/P5/A5）。

## 曾考慮的替代方案

- **逐類獨立擬合精確率曲線**（R1 原案）→ 棄：即 R2 9.2 指出的內傷，小樣本＝噪音。
- **全域單一門檻、不分類**→ 棄：抹平類間真實差異，強類被拖累、弱類被高估；收縮是它與逐類之間的正確折衷。
- **Platt scaling／溫度縮放逐類獨立**→ 棄（作為逐類主力）：同樣吃小樣本噪音；等滲/Platt 可作為 `mapping` 的實作細節，但仍需收縮框住。
- **貝氏階層模型（full MCMC）**→ 棄（現階段）：正確但過重、非確定性、對單機 kernel 過度工程（A10）；`λ_c = N0/(N0+n_c)` 是其閉式退化，先用它。
- **延後校準、先全人審**→ 棄：M0 的老師試鏡與快速佇列正依賴校準分級；沒有它，審核中心退回逐張全審，違背產品前提。

- **觸及條文**：P5（不可驗證的能力視同不存在——reliability 必有快照背書）、P4（人為判斷留痕——low/none 強制人審）、A5（確定性校準）、D5（快照 append-only）、A10（不上 full-Bayes/MCMC，用閉式收縮）、§3 Confidence 分離、R2 §9.2（本 ADR 即其落地）。
